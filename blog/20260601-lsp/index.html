LSP折腾简录·其二

Sukisu Ultra

4.1.3 (40790) 版本似乎支持了越狱模式和软重启，但是如果你的设备已经root了的话，建议别用软重启，似乎会有一些逆天的bug导致系统级优化的lsp模块无法正常启动，所以老老实实完整重启吧。

Play Integrity Fix

好像是过谷歌验证的，通过后可以在Play里显示设备已认证，但还是没法收到一些更新，不太清楚有什么用。

Vector

Vector已经一个月没有更新了，lsp api 102估计也是遥遥无期，不过最后这个版本3043也还算稳定，注意别急着更新需要api 102的模块就是了。

LSPosed

LSP重新发行了，但还是没有公开开源，只有个ksu仓库放了2.0.1和2.0.2两个包，对应api 101和api 102，按需取用吧。

InxLocker

新1.5要求api 102，由于以上原因没法测试了，继续用1.5(44)这个中间版本了。由于一个逆天的bug没法正确锁定，所以需要在作用域勾选可能打开apk的软件。

我不是开发者

担心手机连接至电脑被一条命令攻破？教你一个逆天技巧，给这个模块的作用域勾选system，连接至电脑就除了充电完全没反应了，别说adb，数据传输都不行（）

CamSwap

如果一个模块没推荐勾选系统作用域，那就别勾选。尤其是这种功能的模块，你甚至不知道它会在什么时候发什么电……其实它会在B站播视频时“显示创建渲染：1280*2160”这种东西，但启动其他软件并不会提示什么，很诡异。
